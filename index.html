Step-by-Step Tutorials

Deploy HTML/CSS/JS on Netlify

Step 1: Create Netlify Account

Visit netlify.com and sign up with your GitHub, GitLab, or email account.

Step 2: Prepare Your Project

Ensure your project has an index.html file in the root directory.

<!DOCTYPE html>
<html>
  <head>
    <title>My App</title>
  </head>
  <body>
    <h1>Hello World!</h1>
  </body>
</html>
Step 3: Deploy

Drag and drop your project folder into Netlify dashboard or connect your Git repository.

Step 4: Configure Settings

Set build command (if needed) and publish directory. For static sites, usually no build command is needed.

Step 5: Get Live URL

Netlify will provide a free URL like yoursite.netlify.app. You can customize it or add a custom domain.

Deploy Flutter Web on Firebase Hosting

Step 1: Enable Flutter Web

Make sure Flutter web support is enabled.

flutter config --enable-web
Step 2: Build Web Version

Create a production-ready web build.

flutter build web
Step 3: Install Firebase CLI

Install Firebase command-line tools.

npm install -g firebase-tools
Step 4: Deploy

Initialize Firebase hosting and deploy.

firebase init
firebase deploy

Deploy React on Vercel

Step 1: Install Vercel CLI

Open terminal and install Vercel command-line tool globally.

npm install -g vercel
Step 2: Create React App

Use Vite to create a new React application.

npm create vite@latest my-app -- --template react
cd my-app
npm install
Step 3: Build Your App

Create a production build of your React application.

npm run build
Step 4: Deploy to Vercel

Run the vercel command and follow the prompts.

vercel
Step 5: Access Live Site

Vercel will provide a production URL. Every push to main branch will auto-deploy.

Deploy Node.js on Render

Step 1: Create Render Account

Sign up at render.com with GitHub or email.

Step 2: Prepare Node.js App

Create a basic Express.js server.

const express = require(`express`);
const app = express();
const PORT = process.env.PORT || 3000;

app.get(`/`, (req, res) => {
  res.send(`Hello from Render!`);
});

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});
Step 3: Add package.json

Ensure your package.json has a start script.

{
  "name": "my-app",
  "version": "1.0.0",
  "scripts": {
    "start": "node server.js"
  },
  "dependencies": {
    "express": "^4.18.0"
  }
}
Step 4: Connect Repository

Push code to GitHub and connect it in Render dashboard.

Step 5: Configure Environment

Set build command: npm install. Set start command: npm start. Add environment variables if needed.

Step 6: Deploy

Click deploy and Render will build and host your Node.js app with a free URL.

Deploy Laravel on VPS (Ubuntu + Nginx)

Step 1: Install Required Packages

Install PHP, Composer, and Nginx on your server.

sudo apt update
sudo apt install nginx php php-cli php-fpm php-mysql unzip
Step 2: Install Composer 
curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer
Step 3: Upload Laravel Project

Clone your project from GitHub.

git clone https://github.com/yourname/laravel-app.git
cd laravel-app
composer install
Step 4: Environment Setup

Configure .env and generate app key.

cp .env.example .env
php artisan key:generate
Step 5: Configure Nginx

Point Nginx root to /public directory.

Step 6: Go Live

Your Laravel API is now accessible via your server IP or domain.

Deploy Backend with Docker

Step 1: Create Dockerfile 
FROM node:18
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
CMD ["npm", "start"]
Step 2: Build Docker Image 
docker build -t my-backend .
Step 3: Run Container 
docker run -p 3000:3000 my-backend

Deploy Python Flask on Railway

Step 1: Create Flask App

Create a simple Flask application.

from flask import Flask
import os

app = Flask(__name__)

@app.route(`/`)
def hello():
    return `Hello from Railway!`

if __name__ == `__main__`:
    port = int(os.environ.get(`PORT`, 5000))
    app.run(host=`0.0.0.0`, port=port)
Step 2: Create requirements.txt

List all Python dependencies.

Flask==2.3.0
gunicorn==20.1.0
Step 3: Create Procfile

Tell Railway how to run your app.

web: gunicorn app:app
Step 4: Deploy on Railway

Sign up at railway.app, create new project, and connect your GitHub repository.

Step 5: Configure Settings

Railway auto-detects Python. Add environment variables in settings if needed.

Step 6: Access Your App

Railway provides a public URL. Your Flask app is now live!

Common Deployment Problems

Port Not Open

Make sure firewall allows your app port.

Environment Variables Missing

Always check .env or platform environment settings.

Build Failed

Check logs in deployment dashboard carefully.

Overview

Deployment is the process of making your web application accessible on the internet. It transforms your local development code into a live, publicly accessible website or application.

Why It's Important

  • Makes your work accessible to users worldwide
  • Enables 24/7 availability
  • Provides professional hosting infrastructure
  • Allows for scaling as your user base grows

Real-World Examples

Companies like Netflix, Facebook, and e-commerce sites deploy thousands of times daily using automated CI/CD pipelines.

Key Terminology

Hosting
A service that stores your website files on servers connected to the internet, making them accessible to visitors.
Server
A powerful computer that stores and delivers your web content to users when they visit your site.
Build
The process of converting your source code into optimized files ready for production deployment.
Domain
Your website's address on the internet (e.g., example.com).
DNS (Domain Name System)
Translates human-readable domain names into IP addresses that computers use to locate servers.
HTTP/HTTPS
Protocols for transferring web data. HTTPS is the secure, encrypted version.
Environment Variables
Configuration values stored securely outside your code (API keys, database URLs, etc.).
CI/CD
Continuous Integration/Continuous Deployment - automated testing and deployment processes.
Static Website
Pre-built HTML, CSS, and JavaScript files served directly to users without server processing.
Dynamic Website
Content generated by server-side code based on user requests and database data.
Backend
Server-side application logic, databases, and APIs that power your application.
Frontend
The client-side user interface that users see and interact with in their browsers.
